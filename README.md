# Парсер контактов цветочных магазинов

Скрипт для сбора контактных данных (телефон, email) цветочных магазинов из поисковой выдачи Яндекса.

> Задание выполнено в рамках летней практики в компании **МТС**.

## Что делает скрипт

1. Ищет сайты цветочных магазинов в Яндексе по нескольким городам Беларуси.
2. Заходит на каждый найденный сайт и извлекает телефон и email (при необходимости — заглядывает в раздел «Контакты»).
3. Сохраняет результаты в файл `auto_flower_contacts.csv`.

## Установка

```bash
pip install playwright
playwright install chromium
```

## Запуск

```bash
python main.py
```

При первом запуске откроется окно браузера — при появлении капчи от Яндекса нужно решить её вручную, скрипт продолжит работу автоматически.

## Результат

CSV-файл `auto_flower_contacts.csv` со столбцами:

| Сайт | Телефон | Email |
|------|---------|-------|

# Отчет по итогам создания модели классификации
В ходе работы я успешно создал базовую ML-модель для фильтрации мусорных сайтов. Вот ключевые решения и результаты, к которым я пришел:

## Концентрация на метаданных (Title + Description):
Я подтвердил гипотезу, что сбор метаданных вместо всего текста страницы экономит ресурсы, ускоряет парсинг в 5–10 раз и защищает модель от лишнего текстового шума.

## Выбор алгоритма (Логистическая регрессия):
В качестве отправной точки была выбрана LogisticRegression с параметром class_weight='balanced'. Это позволило автоматически учесть сильный дисбаланс классов в датасете (87% целевых сайтов против 13% мусора) и сильнее штрафовать модель за пропуск мусорных сайтов.

## Борьба с переобучением (Ограничение признаков):
Изначальная попытка использовать 5000 признаков (слов и биграмм) на маленьком датасете привела к тому, что модель просто «зазубрила» данные и выдавала 0% точности на мусорных сайтах. Я снизил количество признаков до 100 самых частых.

## Отказ от биграмм в пользу униграмм (ngram_range=(1, 1)):
Я проверил гипотезу о том, что на небольшом объеме данных словосочетания (биграммы) создают лишний шум. Переход на анализ исключительно одиночных слов позволил модели четко зафиксировать маркеры.

## Итоговый результат:
 - Точность (Precision) класса 0 (Мусор) поднялась до 1.00: Модель теперь безошибочно определяет мусорные сайты (если говорит, что мусор — это 100% так).
 - Полнота (Recall) класса 1 (Цветы) поднялась до 1.00: Модель не заблокировала ни один полезный целевой сайт.
 - Общая точность (Accuracy) системы выросла до 91%.
